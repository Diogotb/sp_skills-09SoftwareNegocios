Atividade: Projeto de Sistema de Biblioteca
Parte 1: Modelagem de Dados
Criação do MER e DER

Crie um Modelo Entidade-Relacionamento (MER) para um sistema de biblioteca. O modelo deve incluir as seguintes entidades:
Livro: Atributos: título, autor, código, ano de publicação.
Usuário: Atributos: nome, matrícula, e-mail.
Empréstimo: Atributos: data de empréstimo, data de devolução.
Defina os relacionamentos entre as entidades:
Um Usuário pode emprestar vários Livros.
Um Livro pode ser emprestado por vários Usuários (muitos para muitos).
Definição de Atributos e Cardinalidade

Para cada entidade, defina os tipos de atributos (simples, composto, multivalorado, derivado).
Especifique a cardinalidade dos relacionamentos.
Parte 2: UML
Diagrama de Casos de Uso

Crie um Diagrama de Casos de Uso para o sistema de biblioteca. Inclua os seguintes atores e casos de uso:
Atores: Usuário, Bibliotecário.
Casos de Uso: Buscar Livro, Reservar Livro, Devolver Livro, Cadastrar Livro.
Desenhe o diagrama, mostrando como os atores interagem com os casos de uso.
Diagrama de Classes

Crie um Diagrama de Classes para o sistema de biblioteca. Inclua as classes correspondentes às entidades do MER e seus métodos.
Defina os relacionamentos entre as classes (associação, agregação, composição).
Parte 3: Consultas em MySQL
Consultas SQL
Crie um banco de dados chamado biblioteca e as tabelas correspondentes às entidades definidas no MER.
Escreva as seguintes consultas SQL:
Inserir Dados: Insira pelo menos 3 livros e 2 usuários na tabela.
Selecionar Dados: Selecione todos os livros disponíveis na biblioteca.
Contar Registros: Conte quantos livros estão cadastrados.
Atualizar Dados: Atualize o título de um livro.
Excluir Dados: Exclua um usuário da tabela.
Parte 4: Relatório
Relatório Final
Elabore um relatório que inclua:
O MER e o DER desenhados.
O Diagrama de Casos de Uso e o Diagrama de Classes.
As consultas SQL escritas e os resultados esperados.
Uma breve descrição do processo de modelagem e das decisões tomadas durante a atividade.
